## 기본 모드

| 모드 | 진입 방법 | 설명 |
|------|-----------|------|
| **Normal** | `Esc` | 이동, 명령 입력 (기본 모드) |
| **Insert** | `i` / `a` / `o` | 텍스트 입력 |
| **Visual** | `v` / `V` / `Ctrl-v` | 텍스트 선택 |
| **Command** | `:` | 명령줄 실행 |

---

## 기본 이동 (Normal 모드)

### 문자/단어 이동

| 키 | 동작 |
|----|------|
| `h` `j` `k` `l` | 좌 / 하 / 상 / 우 |
| `w` | 다음 단어 앞으로 |
| `b` | 이전 단어 앞으로 |
| `e` | 단어 끝으로 |
| `0` | 줄 처음으로 |
| `$` | 줄 끝으로 |
| `^` | 줄의 첫 비공백 문자로 |

### 화면/파일 이동

| 키 | 동작 |
|----|------|
| `gg` | 파일 처음으로 |
| `G` | 파일 끝으로 |
| `{숫자}G` | {숫자}번째 줄로 이동 |
| `Ctrl-d` | 반 화면 아래로 스크롤 |
| `Ctrl-u` | 반 화면 위로 스크롤 |
| `Ctrl-f` | 한 화면 아래로 스크롤 |
| `Ctrl-b` | 한 화면 위로 스크롤 |
| `zz` | 현재 줄을 화면 중앙으로 |

## 기본 편집

### Insert 모드 진입

| 키 | 동작 |
|----|------|
| `i` | 커서 앞에서 입력 |
| `a` | 커서 뒤에서 입력 |
| `I` | 줄 처음에서 입력 |
| `A` | 줄 끝에서 입력 |
| `o` | 아래 줄 새로 추가 후 입력 |
| `O` | 위 줄 새로 추가 후 입력 |

### 삭제 / 복사 / 붙여넣기

| 키 | 동작 |
|----|------|
| `x` | 현재 문자 삭제 |
| `dd` | 현재 줄 삭제 (잘라내기) |
| `dw` | 단어 삭제 |
| `D` | 커서부터 줄 끝까지 삭제 |
| `yy` | 현재 줄 복사 |
| `yw` | 단어 복사 |
| `p` | 커서 뒤에 붙여넣기 |
| `P` | 커서 앞에 붙여넣기 |

### 실행취소 / 재실행

| 키 | 동작 |
|----|------|
| `u` | 실행취소 (Undo) |
| `Ctrl-r` | 재실행 (Redo) |
| `.` | 마지막 명령 반복 |

### 검색 / 치환

| 키 / 명령 | 동작 |
|-----------|------|
| `/패턴` | 앞으로 검색 |
| `?패턴` | 뒤로 검색 |
| `n` | 다음 검색 결과 |
| `N` | 이전 검색 결과 |
| `:%s/찾기/바꾸기/g` | 전체 치환 |
| `Esc Esc` | 검색 하이라이트 제거 |

## LSP (언어 서버)

LSP는 파일을 열면 자동으로 활성화된다.

### 지원 언어

| 언어 | 서버 |
|------|------|
| Lua | `lua_ls` |
| Python | `pyright` |
| TypeScript / JavaScript | `vtsls` |
| C / C++ | `clangd` |
| Rust | `rust_analyzer` |

### LSP 단축키

| 키 | 동작 |
|----|------|
| `gd` | 정의(Definition)로 이동 |
| `gr` | 참조(References) 목록 |
| `K` | 호버 도움말 |
| `<Space>rn` | 이름 변경(Rename) |
| `<Space>ca` | 코드 액션 |
| `<Space>e` | 진단 메시지 팝업 |
| `[d` | 이전 진단으로 이동 |
| `]d` | 다음 진단으로 이동 |

### Mason으로 LSP 서버 관리

```
:Mason          " 설치된 서버 목록 및 관리 UI
:MasonInstall clangd   " 서버 수동 설치
```

## 자동완성 (nvim-cmp)

LSP, 버퍼, 경로, 스니펫에서 자동완성을 제공한다.

| 키 | 동작 |
|----|------|
| `Ctrl-Space` | 완성 목록 강제 표시 |
| `Tab` | 다음 항목 선택 |
| `Shift-Tab` | 이전 항목 선택 |
| `Enter` | 선택 확정 |

## 파일 탐색기 (nvim-tree)

```
:NvimTreeToggle   " 파일 트리 열기/닫기
:NvimTreeFocus    " 파일 트리로 포커스 이동
```

파일 트리 내에서:

| 키 | 동작 |
|----|------|
| `Enter` | 파일 열기 / 폴더 펼치기 |
| `a` | 새 파일/폴더 만들기 |
| `d` | 삭제 |
| `r` | 이름 변경 |
| `q` | 트리 닫기 |

## 퍼지 검색 (Telescope)

```
:Telescope find_files    " 파일명 검색
:Telescope live_grep     " 파일 내용 전체 검색
:Telescope buffers       " 열린 버퍼 목록
:Telescope help_tags     " 도움말 검색
```

Telescope 창 내에서:

| 키 | 동작 |
|----|------|
| `Ctrl-j` / `Ctrl-k` | 목록 위아래 이동 |
| `Enter` | 선택한 파일 열기 |
| `Ctrl-x` | 수평 분할로 열기 |
| `Ctrl-v` | 수직 분할로 열기 |
| `Esc` | 닫기 |

## Gtags 코드 탐색

대규모 C/C++ 코드베이스 탐색에 사용한다. `gtags` 명령으로 DB를 먼저 생성해야 한다.

```bash
# 프로젝트 루트에서 DB 생성
gtags
```

| 키 | 동작 |
|----|------|
| `<Space>ga` | gtags DB 폴더 추가 |
| `<Space>gs` | 커서 단어로 심볼 검색 |
| `<Space>gg` | 커서 단어의 정의 검색 |
| `<Space>gl` | 추가된 DB 목록 출력 |
| `<Space>gc` | DB 목록 초기화 |

## Claude AI 통합

`claude-code.nvim` 플러그인으로 nvim 안에서 Claude Code를 사용한다.  
창은 화면 하단 40% 크기로 수평 분할되어 열린다.

| 키 | 동작 |
|----|------|
| `<Space>ac` | Claude Code 창 열기/닫기 |
| `<Space>ar` | 이전 세션 재개 |
| `<Space>aC` | 이전 대화 이어서 계속 |

## 색상 테마

**tokyonight** 테마를 사용하며, 터미널 투명 배경이 적용된다.

- 검색 결과: 노란 배경 (`#ffff00`)
- 현재 검색 위치: 주황 배경 (`#ff8800`)

## 기타 설정 요약

| 항목 | 값 |
|------|----|
| 탭 크기 | 8칸 (실제 탭 문자 사용) |
| 줄 번호 | 절대 번호 표시 |
| 마우스 | 기본 비활성 (`<Space>tm`으로 켜기) |
| 클립보드 | 시스템 클립보드 미연동 (기본) |
| 창 분할 방향 | 오른쪽 / 아래 |
| 스크롤 여백 | 8줄 (`scrolloff=8`) |
| 마지막 편집 위치 | 파일 재열기 시 자동 복원 |
| 구문 강조 | Treesitter (Lua, Python, JS/TS, Bash, Vim) |
